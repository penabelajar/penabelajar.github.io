2. cara mengatasi the specified network name is no longer available windows windows server 2012


Anda mungkin tidak dapat mengakses saham pada mesin klien pada salah satu keadaan berikut ini:

 

1. ketika Anda mencoba membuka berbagi dari mesin atau mencoba memetakan kandar, itu


menjadi sangat lambat, memerlukan banyak waktu untuk membuka atau tidak tersambung. Anda mungkin menerima galat berikut ini:

"Nama jaringan yang dimaksud sudah tidak lagi tersedia"

 

2. \\domain.local atau \\servername atau \\IPAddress tidak membawa berbagi

server itu sendiri.  Anda mungkin menerima galat berikut ini:

 

"Nama jaringan yang dimaksud sudah tidak lagi tersedia"

 

 

3. DCDIAG mungkin gagal di berbagai tes.  Anda mungkin menerima galat berikut ini:

 


"gagal dengan 64: nama jaringan yang dimaksud tidak tersedia"


 

4. Anda tidak dapat menjalankan netdom untuk me-reset saluran aman menggunakan netdom /resetpwd.   Anda mungkin menerima galat berikut ini:

 


"Nama jaringan yang dimaksud sudah tidak lagi tersedia"

 


5. dalam kasus jika Server (DC) DHCP server, klien mungkin tidak mendapatkan konfigurasi IP dari server sebaliknya, mereka mungkin akan menerima alamat representasi-alamat IP pribadi otomatis (APIPA), seperti 169.254.0.1.

 

6. Anda mungkin menerima id peristiwa 1058 berikut ini:

 

Jenis peristiwa: kesalahan


Sumber peristiwa: Userenv

Kategori Peristiwa: Tidak Ada

ID Kejadian: 1058

Tanggal: 1/22/2008

Waktu: 11:54:27: 00

Pengguna: NT AUTHORITY\SYSTEM

Keterangan:

Windows tidak dapat mengakses berkas gpt.ini untuk GPO CN = {31B2F340 - 016D - 11D 2 - 945F -


00C04FB984F9}, CN = Policies, CN = System, DC = WTMe dia, DC = local. Berkas harus ada

di lokasi < \\domain.local\sysvol\domain.local\Polic


ies\ {31B2F340-016D-11D2-945F-00C04FB984F9} \gpt.ini >. (Informasi konfigurasi

tidak dapat dibaca dari kontroler domain, baik karena mesin

tidak tersedia, atau akses ditolak. ). Pemrosesan kebijakan grup dibatalkan.

3. jawab dengan baik dan benar cara mengatasi koneksi wifi yang limited access di windows 7


Klw wifi pasang sendiri di rumah, biasanya harus dimatikan dulu baru dihidupkan lagi.Tiap Operating System memiliki kelebihan dan kekurangan masing-masing, Windows 7 disukai pengguna karena tampilannya yang menarik, namun di balik itu banyak thread di internet yang mengeluhkan bahwa Windows 7 sering trouble di koneksi internetnya, kalau tidak "limited or no connectivity" atau sudah "connected" tapi " no internet access". hal ini pun sering di jumpai di Hotel Grand Pangestu, tamu sering menayakan mengenai password Wifi, padahal di Hotel Grand Pangestu koneksi internetnya tidak menggunakan Password, tamu yang sering kesulitan biasanya menggunakan OS win 7, berikut kami berikan cara yang mungkin dapat menyelesaikan masalah koneksi di Windows 7;
Langkah-langkah;
Pastikan perangkat WiFi sudah "on" 
1. Arahkan kursor ke icon wireless agent yang ada di pojok kanan bawah, lalu klik kanan, lalu pilih trouble shooter ( biarkan sistem yang mencari dan memperbaikinya). jika sistem telah memperbaikinya cobalah browsing, jika belum ikuti langkah 2
2. arahkan kursor ke icon wireless agent, klik kanan, pilih "Properties", lalu pilih "Ipv4" lalu IP Config buat "Automatic" termasuk DNS config 
3. Masuklah ke Control Panel, lalu pilih "Administrative tools", kemudian pilih "Services"  kemudian dalam window service cari menu "DHCP Cient", "DNS Client",dan "WLAN Auto Config", ketiga menu ini harus di restart satu persatu, jika ketiga menu ini sudah di restart coba anda lihat icon wireless agent jika sudah tidak ada tanda seru, coba browsing, demikian cara yang sederhana ini semoga membantu

4. cara mengatasi Windows 7 tidak support ​


Jawaban:

Menghilangkan Pesan “Your Windows 7 PC is Out of Support”

Buka Registry Editor.

Navigasi ke path HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\EOSNotify.

Pada panel kanan, klik kanan dan pilih New – DWORD (32-bit). Kemudian beri nama DiscontinueEOS.

Klik 2x pada key baru tersebut, lalu masukan value 1 (pastikan kamu menggunakan opsi Decimals). ...

Done.
